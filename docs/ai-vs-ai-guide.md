# AI-vs-AI Debates — End User Guide

Run a fully autonomous IPDA debate where both sides are generated by the LLM. You create the debate, open a WebSocket, and observe — no human input required.

## When to use this

- You want to watch the AI debate itself for evaluation, data collection, or entertainment.
- You're building a spectator UI (stream the speeches as they're generated).
- You want to benchmark the model against itself on different topics.
- You need to produce training data or debate transcripts at scale.

For human-vs-AI debates, use `POST /debates/managed` instead — see the main [README](../README.md).

## TL;DR — the full call

```bash
# 1. Create the debate (returns instantly)
curl -X POST http://localhost:8000/debates/ai-vs-ai \
  -H "Content-Type: application/json" \
  -d '{"topic": "The United States should adopt universal basic income"}'
# → {"debate_id": "a126bf4a622f", "status": "creating", ...}

# 2. Connect the WebSocket observer (see Python snippet below)
# 3. Events stream in for ~20–25 minutes until judge_result arrives
```

## Endpoint reference

### `POST /debates/ai-vs-ai`

Creates an autonomous AI-vs-AI debate. Non-blocking — returns the `debate_id` immediately while setup runs in the background.

**Request body**
```json
{
  "topic": "The United States should adopt universal basic income"
}
```

Or reference a pre-built topic (belief tree already cached):
```json
{
  "topic_id": "f1a2b3c4d5e6"
}
```

One of `topic` or `topic_id` is required. See [Topic reuse](#topic-reuse) below.

**Response (instant)**
```json
{
  "debate_id": "a126bf4a622f",
  "topic_id": null,
  "topic": "The United States should adopt universal basic income",
  "status": "creating",
  "message": "AI-vs-AI debate is being created. Connect via WebSocket to observe."
}
```

**What happens server-side:**
1. A `debate_id` is reserved immediately (you can open the WebSocket now — events buffer).
2. The Modal agent container is warmed up (poll status sent as `warmup_progress` events).
3. A LiveKit room is created and the debate agent is dispatched with `debate_mode=ai_ai`.
4. The agent runs both sides of the debate autonomously over ~20–25 minutes.

## Observing the debate

Connect the WebSocket **immediately** after the POST — events buffer during setup and flush once the agent is ready.

```
ws://localhost:8000/debates/{debate_id}/ws
```

### Minimal Python observer

```python
import asyncio
import json
import httpx
import websockets

BASE_URL = "http://localhost:8000"
TOPIC = "The United States should adopt universal basic income"

async def run_ai_debate():
    # 1. Create the debate
    async with httpx.AsyncClient() as http:
        resp = await http.post(
            f"{BASE_URL}/debates/ai-vs-ai",
            json={"topic": TOPIC},
            timeout=30.0,
        )
        resp.raise_for_status()
        debate_id = resp.json()["debate_id"]
        print(f"Created debate {debate_id}")

    # 2. Connect WebSocket and observe (no actions sent)
    ws_url = f"ws://localhost:8000/debates/{debate_id}/ws"
    async with websockets.connect(ws_url) as ws:
        while True:
            raw = await asyncio.wait_for(ws.recv(), timeout=1800)  # 30 min max wait
            event = json.loads(raw)
            etype = event.get("type")

            if etype == "warmup_progress":
                print(f"  warming: {event['agent_status']}")
            elif etype == "debate_initializing":
                print(f"  initializing: {event['message']}")
            elif etype == "debate_ready":
                print(f"  ready! speech order: {event['speech_order']}")
            elif etype == "turn_signal":
                print(f"  → {event['speech_type']} ({event['speaker']}) — {event['status']}")
            elif etype == "speech_text":
                print(f"  ✓ {event['speech_type']} speech: {event['word_count']} words")
            elif etype == "cx_question":
                print(f"  ? CX question: {event['question'][:80]}")
            elif etype == "judge_result":
                print(f"\nWINNER: {event['winner']}")
                print(f"  aff={event['aff_score']} neg={event['neg_score']}")
                print(f"  margin={event['margin']}")
                return event
            elif etype == "error":
                print(f"  ERROR: {event['message']}")
                if not event.get("recoverable", True):
                    raise RuntimeError(event["message"])

asyncio.run(run_ai_debate())
```

### Event sequence you will see

A successful AI-vs-AI debate produces ~200+ events in this order:

```
warmup_progress          (×N while Modal container spins up)
debate_initializing      ← topic + message, agent is starting
belief_tree              ← argument map for both sides
debate_ready             ← speech_order + time limits
 ┌── per speech ──┐
 │ turn_signal    (status=active)
 │ speech_progress  (stage=tactic)
 │ speech_progress  (stage=skeleton)
 │ speech_progress  (stage=evidence)
 │ speech_progress  (stage=generate)
 │ speech_text      ← full speech transcript
 │ flow_update      ← updated argument flow
 │ (CX period: cx_question / cx_answer pairs)
 └────────────────┘
judging_started
judge_result             ← winner, scores, feedback
```

All 5 constructive/rebuttal speeches (`AC, NC, 1AR, NR, 2AR`) emit `speech_text` events. The 2 CX periods (`AC-CX, NC-CX`) emit a series of `cx_question`/`cx_answer` events instead.

## REST fallbacks

If your client can't hold a WebSocket open, use polling:

```bash
# Current state
curl http://localhost:8000/debates/{debate_id}/status
# → {"status": "active", "current_speech": "NC", "is_complete": false, ...}

# Full event history (everything that has happened so far)
curl http://localhost:8000/debates/{debate_id}/events

# Just speech_text events
curl "http://localhost:8000/debates/{debate_id}/events?event_type=speech_text"

# New events since a timestamp (for catch-up polling)
curl "http://localhost:8000/debates/{debate_id}/events?since=1776188000"

# All completed transcripts (once speeches have been generated)
curl http://localhost:8000/debates/{debate_id}/transcripts

# The belief tree (argument map for both sides)
curl http://localhost:8000/debates/{debate_id}/belief-tree
curl http://localhost:8000/debates/{debate_id}/belief-tree/aff  # aff only
curl http://localhost:8000/debates/{debate_id}/belief-tree/neg  # neg only
```

These work both during and after the debate — perfect for dashboards or late-joining observers.

## Topic reuse

A reusable topic caches the belief tree so subsequent debates on the same resolution skip the ~30 second prep phase.

```bash
# 1. Create the topic (returns instantly)
curl -X POST http://localhost:8000/debates/topics \
  -H "Content-Type: application/json" \
  -d '{"topic": "The United States should adopt universal basic income"}'
# → {"topic_id": "f1a2b3c4d5e6", "status": "building", "has_belief_tree": false}

# 2. Poll until ready (tree builds in the background, ~30s–30min)
curl http://localhost:8000/debates/topics/f1a2b3c4d5e6
# → {"status": "ready", "has_belief_tree": true, "debate_count": 0}

# 3. Create N debates against the cached topic
curl -X POST http://localhost:8000/debates/ai-vs-ai \
  -H "Content-Type: application/json" \
  -d '{"topic_id": "f1a2b3c4d5e6"}'
```

You don't have to wait for `ready` — starting a debate on a `building` topic works fine, it just won't skip the prep phase.

## Running many debates in parallel

Each debate is independent. Fire them off concurrently:

```python
import asyncio
import httpx

TOPICS = [
    "UBI should replace existing welfare programs",
    "AI systems should be required to pass safety audits",
    "Social media platforms should be liable for user content",
]

async def create_and_observe(topic: str):
    async with httpx.AsyncClient() as http:
        r = await http.post(
            "http://localhost:8000/debates/ai-vs-ai",
            json={"topic": topic}, timeout=30
        )
        debate_id = r.json()["debate_id"]
        # ... open WebSocket, observe until judge_result ...
        return debate_id

async def main():
    results = await asyncio.gather(*[create_and_observe(t) for t in TOPICS])
    print("Completed:", results)

asyncio.run(main())
```

The Modal agent scales out, but be mindful of concurrent container cost.

## Common issues

**"Only `warmup_progress` events, nothing else arrives"**
The Modal agent container is still cold-starting. First debate after a period of inactivity can take 60–90 seconds before `debate_initializing` appears. Check the warmup endpoint directly:
```bash
curl https://debaterhub--livekit-human-debate-agent-warmup.modal.run
# → {"status": "ready", "containers": 1}
```

**"Got an error event with `recoverable: false`"**
Setup failed — check `message` and `code`. Common causes: LiveKit credentials wrong, agent not deployed, topic too ambiguous for prep.

**"WebSocket connection closes mid-debate"**
The debate continues running server-side. Reconnect the same `debate_id` or poll `/events` to catch up — the event history endpoint returns everything you missed.

**"How do I know the debate is actually done?"**
Wait for `judge_result` on the WebSocket, or poll `GET /debates/{id}/status` until `is_complete: true`.

## What you get at the end

When `judge_result` arrives, pull these for your UI/storage:

```bash
# The winner and scores
curl http://localhost:8000/debates/{id}/events?event_type=judge_result

# All 5 speech transcripts
curl http://localhost:8000/debates/{id}/transcripts

# The argument map
curl http://localhost:8000/debates/{id}/belief-tree

# Full event log (for reconstruction / replay)
curl http://localhost:8000/debates/{id}/events
```

A full AI-vs-AI IPDA debate is ~10,000–15,000 words of generated speech text across the 5 constructive/rebuttal speeches plus CX exchanges.
